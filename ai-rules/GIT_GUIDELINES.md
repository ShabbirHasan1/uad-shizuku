## Conventional Commit Formats

```
<type>[optional scope]: <description>

    <type>: Mandatory. Specifies the kind of change (e.g., feat, fix, docs, style, refactor, test, chore).
    [optional scope]: Optional. Describes the part of the codebase that was affected (e.g., core, auth, ui).
    <description>: Mandatory. A short description of the change, following the general best practices listed above. 

Example:
feat(api). add new endpoint for user authenticatio
```
