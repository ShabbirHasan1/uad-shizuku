# 사용법

이 가이드는 UAD-Shizuku를 사용하여 Android 기기의 애플리케이션을 관리하는 방법을 설명합니다.

## 시작하기

### 사전 요구사항

1. **ADB (Android Debug Bridge)**: Android 기기에 연결하는 데 필요합니다
   - 처음 실행 시 ADB를 찾을 수 없으면 설치를 안내하는 대화상자가 표시됩니다
   - 데스크톱 플랫폼(Windows/Linux/macOS)에서는 앱이 자동으로 platform-tools를 다운로드할 수 있습니다

2. **USB 디버깅**: Android 기기에서 USB 디버깅을 활성화해야 합니다
   - 설정 > 개발자 옵션 > USB 디버깅으로 이동
   - USB로 기기를 연결하거나 무선 ADB를 설정

3. **Shizuku (선택사항)**: Android APK 모드에서 Shizuku는 루트 없이 루트와 유사한 접근 권한을 제공합니다

### 초기 설정

1. UAD-Shizuku 실행
2. 면책 조항 대화상자 수락
3. USB 디버깅이 활성화된 Android 기기 연결
4. **기기** 드롭다운에서 기기 선택
5. **사용자** 드롭다운에서 사용자 선택 (또는 "모든 사용자"로 유지)
6. 앱이 설치된 패키지 목록을 로드합니다

## 탭

UAD-Shizuku에는 네 가지 주요 탭이 있습니다:

### 1. 디블로트 탭

Android 기기에서 블로트웨어를 관리하고 제거합니다.

#### 필터 카테고리

패키지는 [UAD-NG 블로트 목록](https://github.com/Universal-Debloater-Alliance/universal-android-debloater-next-generation)을 기반으로 분류됩니다:

| 카테고리 | 색상 | 설명 |
|----------|------|------|
| **권장** | 녹색 | 제거해도 안전, 일반적으로 블로트웨어 |
| **고급** | 파란색 | 약간의 지식 필요, 일부 기능에 영향을 줄 수 있음 |
| **전문가** | 주황색 | 기능이 손상될 수 있음, 주의해서 사용 |
| **안전하지 않음** | 노란색 | 높은 위험, 시스템 불안정을 유발할 수 있음 |
| **알 수 없음** | 흰색 | UAD-NG 데이터베이스에 없음 |

#### 패키지 작업

각 패키지에 대해 다음 작업을 수행할 수 있습니다:

- **정보** : 상세한 패키지 정보 보기
- **제거** : 패키지 제거 (사용자 앱) 또는 현재 사용자에 대해 제거 (시스템 앱)
- **비활성화** : 제거하지 않고 패키지 비활성화
- **활성화** : 비활성화되거나 제거된 패키지 다시 활성화

#### 일괄 작업

1. 체크박스를 사용하여 여러 패키지 선택
2. 일괄 작업 버튼 사용:
   - **선택 항목 제거**: 선택한 모든 패키지 제거
   - **선택 항목 비활성화**: 선택한 모든 패키지 비활성화
   - **선택 항목 활성화**: 선택한 모든 패키지 활성화
3. **모두 선택 해제**를 클릭하여 선택 취소

#### 필터 및 옵션

- **활성화된 항목만 표시**: 비활성화/제거된 패키지 숨기기
- **시스템 앱 숨기기**: 시스템 애플리케이션 숨기기
- **필터**: 패키지 이름, 카테고리, 앱 정보에서 텍스트 검색

---

### 2. 스캔 탭

온라인 서비스를 사용하여 설치된 애플리케이션의 보안 위협을 스캔합니다.

#### 스캔 서비스

| 서비스 | 설명 |
|--------|------|
| **VirusTotal** | 다중 안티바이러스 스캐너 (API 키 필요) |
| **Hybrid Analysis** | 샌드박스 기반 악성코드 분석 (API 키 필요) |
| **IzzyRisk** | 앱 권한 및 동작 기반 위험 점수 |

#### 스캔 필터

스캔 결과로 패키지 필터링:
- **전체**: 모든 패키지 표시
- **악성**: 악성코드로 표시된 패키지
- **의심스러움**: 의심스러운 지표가 있는 패키지
- **안전**: 스캔 결과가 깨끗한 패키지
- **스캔되지 않음**: 스캔 대기 중인 패키지

#### 스캔 방법

1. 설정에서 API 키 설정 (VirusTotal 및 Hybrid Analysis용)
2. 스캔 탭으로 이동
3. 패키지가 로드될 때 자동으로 스캔됨
4. 알림 영역에서 스캔 진행 상황 확인
5. 진행 중인 스캔을 취소하려면 **중지** 클릭

---

### 3. 앱 탭

FOSS(자유 오픈소스 소프트웨어) 애플리케이션을 탐색하고 설치합니다.

#### 앱 소스

- **OFFA**: [android-foss](https://github.com/offa/android-foss) 큐레이션 목록의 앱
- **FMHY**: [FMHY 모바일 가이드](https://fmhy.pages.dev/mobile)의 앱

#### 앱 설치

1. 드롭다운에서 앱 소스 선택
2. 앱 탐색 또는 검색
3. 앱 카드의 설치 버튼 클릭
4. ADB를 통해 앱 다운로드 및 설치

#### 옵션

- **설치 가능한 항목만 표시**: 다운로드 링크가 있는 앱으로 필터링 (F-Droid, GitHub 등)
- **GitHub 설치 비활성화**: GitHub 릴리스를 다운로드 소스에서 제외

## 설정

메뉴 > 설정을 통해 설정에 접근합니다.

### 외관

| 설정 | 옵션 | 설명 |
|------|------|------|
| **언어** | 영어, 한국어 | 애플리케이션 인터페이스 언어 |
| **글꼴** | 시스템 글꼴 | 인터페이스용 사용자 지정 글꼴 |
| **테마** | 라이트, 다크, 자동 | 색상 테마 모드 |
| **대비** | 보통, 중간, 높음 | UI 대비 수준 |
| **텍스트 스타일** | 작게, 본문, 버튼, 제목, 고정폭 | 텍스트 크기 재정의 |

### API 키

| 설정 | 설명 |
|------|------|
| **VirusTotal API 키** | VirusTotal 스캔용 API 키 ([키 받기](https://www.virustotal.com/gui/my-apikey)) |
| **Hybrid Analysis API 키** | Hybrid Analysis용 API 키 ([키 받기](https://www.hybrid-analysis.com/my-account?tab=%23api-key-tab)) |

### 렌더러

외부 소스에서 앱 정보 가져오기 활성화:

| 렌더러 | 설명 |
|--------|------|
| **Google Play** | Google Play 스토어에서 앱 정보 가져오기 |
| **F-Droid** | F-Droid 저장소에서 앱 정보 가져오기 |
| **APKMirror** | APKMirror에서 앱 정보 가져오기 (시스템 앱용) |

### 스캔 옵션

| 설정 | 설명 |
|------|------|
| **VirusTotal 제출** | VirusTotal 데이터베이스에 없는 APK 자동 제출 |
| **Hybrid Analysis 제출** | Hybrid Analysis 데이터베이스에 없는 APK 자동 제출 |
| **HA 태그 블랙리스트** | Hybrid Analysis 결과에서 무시할 쉼표로 구분된 태그 |

### 로깅

| 설정 | 설명 |
|------|------|
| **로그 표시** | 창 하단에 로그 패널 표시 |
| **로그 수준** | Error, Warn, Info, Debug, Trace |

### 캐시 관리

| 작업 | 설명 |
|------|------|
| **캐시 무효화** | 로컬 패키지 캐시 삭제 |
| **VirusTotal 초기화** | VirusTotal 스캔 결과 삭제 |
| **Hybrid Analysis 초기화** | Hybrid Analysis 스캔 결과 삭제 |
| **Google Play 초기화** | Google Play 앱 정보 캐시 삭제 |
| **F-Droid 초기화** | F-Droid 앱 정보 캐시 삭제 |
| **APKMirror 초기화** | APKMirror 앱 정보 캐시 삭제 |

## 문제 해결

### ADB를 찾을 수 없음

1. ADB 설치 대화상자에서 **탐지 재시도** 클릭
2. ADB를 여전히 찾을 수 없으면 대화상자의 플랫폼별 설치 지침을 따르세요
3. 수동 설치의 경우 ADB가 시스템 PATH에 있는지 확인하세요

### 기기가 감지되지 않음

1. 기기에서 USB 디버깅이 활성화되어 있는지 확인
2. 기기가 올바르게 연결되어 있는지 확인
3. **새로고침** 버튼을 클릭하여 기기 다시 검색
4. 기기에서 USB 디버깅 권한 요청 수락

### 패키지 목록이 비어 있음

1. 기기 연결 확인
2. 선택된 사용자 확인 ("모든 사용자" 시도)
3. 로그 패널에서 오류 메시지 확인 (설정에서 활성화)

### 스캔 오류

1. API 키가 올바른지 확인
2. 인터넷 연결 확인
3. API 서비스의 요청 제한 검토
4. 로그 패널에서 상세 오류 메시지 확인

## 플랫폼 참고사항

### 데스크톱 (Windows/Linux/macOS)

- 메뉴 > 설치를 통한 시스템 설치 지원
- 새 릴리스에 대한 자동 업데이트 확인
- ADB platform-tools 자동 다운로드

### Android (APK)

- 패키지 관리를 위해 Shizuku 필요
- 다른 기기에 대한 무선 ADB 연결
- 네이티브 Android UI 경험
